# Portfolio Prototype Directions

These files are static HTML prototypes for comparing portfolio directions before changing the Next.js app.

## Files

- `portfolio-redesign-sample.html`
  - Revised baseline.
  - Big-name hero, editorial project list, writing/contact sections.

- `portfolio-style-01-index-archive.html`
  - Portfolio-native archive.
  - Feedback: too basic/default as a final direction, but useful as a structural reference.

- `portfolio-style-02-notebook.html`
  - Personal notebook direction.
  - Feedback: rejected.

- `portfolio-style-03-visual-dossier.html`
  - More graphic and image-led.
  - Feedback: selected final direction.
  - Good base if final project screenshots are strong and visual polish matters most.

- `portfolio-style-04-color-block-archive.html`
  - Loud color-block portfolio.
  - Best if the final site should feel playful, bold, and memorable.

- `portfolio-style-05-swiss-editorial.html`
  - Mature editorial portfolio.
  - Less default than style 1, less loud than style 4.

- `portfolio-style-06-poster-wall.html`
  - Dark poster-wall portfolio.
  - Closest to a creative gallery while still keeping developer content clear.

- `portfolio-style-07-technical-ledger.html`
  - Developer-native ledger.
  - More designed than a plain archive, but still technical and structured.

- `portfolio-style-08-image-index.html`
  - Image-led index.
  - Softer cousin of style 3 with bigger visual records.

- `portfolio-style-09-3d-workbench.html`
  - Playful 3D workbench direction.
  - Closest to the Joshua Comeau-style ask: personal, interactive-feeling, and not product-site-like.
  - Prototype uses CSS 3D so it stays standalone; the real app should use Three.js if this direction is selected.

- `portfolio-style-10-generative-lab.html`
  - Generative canvas lab wall.
  - More experimental and atmospheric, with animated particles behind clear portfolio content.

- `portfolio-style-11-inventory.html`
  - Game/inventory-inspired portfolio.
  - Most playful option; project entries become inspectable “items” instead of cards.

## Current Recommendation

Use `portfolio-style-03-visual-dossier.html` as the selected direction for implementation. The dedicated implementation plan is `docs/plans/2026-06-18-visual-dossier-implementation-plan.md`.
